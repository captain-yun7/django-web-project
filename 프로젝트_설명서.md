# 웹 게시판 프로젝트 설명서
> 비기술자를 위한 쉬운 설명

## 📌 이 프로젝트가 무엇인가요?

이 프로젝트는 **인터넷 게시판 웹사이트**입니다.
다음과 같은 기능들을 사용할 수 있습니다:

- ✅ 회원가입하고 로그인하기
- ✅ 글 작성하고 파일 첨부하기
- ✅ 다른 사람의 글 읽기
- ✅ 댓글 달기
- ✅ 글 검색하기
- ✅ 내가 쓴 글 수정/삭제하기

마치 **네이버 카페**나 **에브리타임** 같은 커뮤니티 게시판을 만든 것입니다.

---

## 🖥️ 프로젝트 구성

이 웹사이트는 크게 3개 부분으로 나뉩니다:

### 1️⃣ 화면 (프론트엔드)
> 사용자가 보는 웹페이지 부분

- **기술**: React (리액트)
- **역할**:
  - 예쁜 화면 디자인
  - 버튼 클릭, 글 작성 등의 동작 처리
  - 사용자와 상호작용하는 모든 것
- **비유**: 식당의 홀 - 손님이 직접 보고 주문하는 공간

### 2️⃣ 서버 (백엔드)
> 데이터를 처리하는 두뇌 부분

- **기술**: Django (장고)
- **역할**:
  - 회원가입, 로그인 처리
  - 글 작성, 수정, 삭제 처리
  - 파일 업로드 처리
  - 보안 관리 (비밀번호 암호화 등)
- **비유**: 식당의 주방 - 실제 요리를 만드는 곳

### 3️⃣ 데이터베이스
> 모든 정보를 저장하는 창고

- **기술**: MySQL (마이에스큐엘)
- **저장하는 것**:
  - 회원 정보 (이메일, 비밀번호)
  - 게시글 (제목, 내용, 작성자)
  - 댓글
  - 첨부파일 정보
- **비유**: 식당의 창고 - 재료를 보관하는 곳

## 🎯 주요 기능 설명

### 1. 회원가입 & 로그인

**어떻게 작동하나요?**
1. 사용자가 이메일과 비밀번호를 입력합니다
2. 서버가 비밀번호를 암호화해서 데이터베이스에 저장합니다
3. 로그인 시 입력한 정보와 저장된 정보를 비교합니다
4. 맞으면 "출입증(토큰)"을 발급해서 30분간 사용할 수 있게 합니다

**보안 기능**:
- 비밀번호 5번 틀리면 30분 동안 로그인 차단 (해킹 방지)
- 비밀번호는 암호화되어 저장 (직원도 못 봅니다)

### 2. 게시글 작성

**어떻게 작동하나요?**
1. 로그인한 사용자만 글을 쓸 수 있습니다
2. 제목과 내용을 입력합니다
3. 필요하면 파일을 첨부합니다 (최대 10MB)
4. "등록" 버튼을 누르면:
   - 먼저 글이 저장됩니다
   - 그 다음 파일이 업로드됩니다
   - 새로 쓴 글로 자동 이동합니다

**첨부 가능한 파일**:
- 문서: PDF, Word, Excel, PowerPoint
- 이미지: JPG, PNG, GIF
- 압축파일: ZIP

### 3. 댓글 작성

**어떻게 작동하나요?**
1. 글 아래에 댓글 입력창이 있습니다
2. 댓글을 입력하고 "등록" 버튼을 누릅니다
3. 즉시 화면에 표시됩니다
4. 본인이 쓴 댓글은 "삭제" 버튼으로 지울 수 있습니다

### 4. 검색

**어떻게 작동하나요?**
1. 상단 검색창에 단어를 입력합니다
2. 제목과 내용에서 해당 단어가 포함된 글을 찾습니다
3. 검색 결과를 바로 보여줍니다

### 5. 파일 다운로드

**어떻게 작동하나요?**
1. 글 하단에 "첨부파일" 섹션이 표시됩니다
2. 파일 이름을 클릭하면 다운로드됩니다
3. 파일 크기도 함께 표시됩니다 (예: 2.6 KB)

## 🚀 프로젝트 실행 방법

### 방법 1: 자동 설치 (추천)

**가장 간단한 방법입니다!**

```bash
# 1. 프로젝트 폴더로 이동
cd django-web-project

# 2. 아래 명령어 하나만 실행하면 끝!
./auto-setup-wsl.sh
```

이 스크립트가 자동으로:
- ✅ Docker 설치 (아직 없다면)
- ✅ 필요한 설정 파일 생성
- ✅ 웹사이트 실행
- ✅ 데이터베이스 준비

**5~10분 정도 기다리면 완성!**

### 방법 2: 수동 설치

이미 Docker가 설치되어 있다면:

```bash
docker compose up -d
```

## 📱 사용 방법 (단계별)

### 1단계: 회원가입

1. http://localhost:3000/register 접속
2. 이메일 주소 입력 (예: user@example.com)
3. 비밀번호 입력
4. "회원가입" 버튼 클릭

### 2단계: 로그인

1. http://localhost:3000/login 접속
2. 가입한 이메일, 비밀번호 입력
3. "로그인" 버튼 클릭

### 3단계: 글쓰기

1. 로그인 후 "글쓰기" 버튼 클릭
2. 제목 입력
3. 내용 입력
4. (선택) 파일 첨부 - 여러 개 가능!
5. "등록" 버튼 클릭

### 4단계: 댓글 달기

1. 글을 클릭해서 상세 페이지로 이동
2. 아래쪽 댓글 입력창에 내용 입력
3. "등록" 버튼 클릭

### 5단계: 검색하기

1. 상단 검색창에 찾고 싶은 단어 입력
2. 자동으로 결과 표시

## 🔒 보안 기능

### 1. 로그인 보호
- 비밀번호 5번 틀리면 → 30분 차단
- 해커의 무차별 대입 공격 방지

### 2. 비밀번호 암호화
- 비밀번호를 암호화해서 저장
- 관리자도 원본 비밀번호를 볼 수 없음

### 3. 파일 업로드 제한
- 위험한 파일 차단 (.exe, .bat 등)
- 파일 크기 제한 (최대 10MB)
- 서버 과부하 방지

### 4. 권한 관리
- 본인 글만 수정/삭제 가능
- 본인 댓글만 삭제 가능
- 로그인 안 하면 글 작성 불가

## ❓ 자주 묻는 질문 (FAQ)

### Q1. 설치가 안 돼요!
**A**: 다음을 확인해보세요:
1. Docker가 실행 중인지 확인
2. 터미널에서 `sudo dockerd &` 실행 (WSL인 경우)
3. 스크립트 실행 권한: `chmod +x auto-setup-wsl.sh`

### Q2. 로그인이 안 돼요!
**A**: 다음을 확인해보세요:
1. 회원가입을 먼저 했는지 확인
2. 이메일과 비밀번호를 정확히 입력했는지 확인
3. 5번 이상 틀렸다면 30분 후 다시 시도

### Q3. 파일 업로드가 안 돼요!
**A**: 다음을 확인해보세요:
1. 파일 크기가 10MB 이하인지 확인
2. 허용된 파일 형식인지 확인 (pdf, doc, jpg 등)
3. 로그인이 되어 있는지 확인

### Q4. 화면이 이상해요!
**A**: 브라우저 새로고침:
- Windows: `Ctrl + Shift + R`
- Mac: `Cmd + Shift + R`

### Q5. 데이터를 모두 지우고 싶어요!
**A**: 다음 명령어 실행:
```bash
docker compose down -v
```
**주의**: 모든 게시글, 회원정보가 삭제됩니다!

## 📞 문제 해결

### 상황 1: "Docker가 실행되지 않습니다"
```bash
# WSL 환경
sudo dockerd &

# 일반 Ubuntu
sudo systemctl start docker
```

### 상황 2: "포트가 이미 사용 중입니다"
```bash
# 기존 컨테이너 중지
docker compose down

# 다시 시작
docker compose up -d
```

### 상황 3: "화면이 안 나와요"
1. 백엔드 확인: http://localhost:8000/api/
2. 프론트엔드 확인: http://localhost:3000
3. 둘 다 안 되면:
```bash
docker compose logs backend
docker compose logs frontend
```

### 상황 4: "관리자 계정이 필요해요"
```bash
docker compose exec backend python manage.py createsuperuser
```
그 다음 http://localhost:8000/admin 접속
